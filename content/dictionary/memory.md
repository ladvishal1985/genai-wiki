Memory in the context of AI agents refers to mechanisms for persisting and retrieving information across turns, sessions, or agent steps beyond what fits in the active context window. Short-term memory is the context window itself; long-term memory involves external stores such as vector databases (for semantic retrieval), key-value stores (for structured facts), or episodic logs (for conversation history). Agents must decide what to store, when to retrieve, and how to integrate retrieved memories with the current task. Effective memory management is essential for personalised assistants, long-running task agents, and systems that need to learn from past interactions.
