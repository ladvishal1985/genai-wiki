Working memory in LLM agents refers to the agent's active reasoning state—the intermediate computations, partial results, and current task context held during problem-solving. In practice, this is the scratch-pad portion of the context window used for chain-of-thought, tool call results being processed, and sub-task tracking. Unlike long-term memory, working memory is temporary and discarded between tasks. Reasoning tokens in extended thinking models can be seen as explicit working memory.
